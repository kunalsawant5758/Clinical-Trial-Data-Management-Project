# Clinical Trial Data Management Project

## Overview

This project was developed to simulate key Clinical Data Management (CDM) activities performed during clinical trials. A simulated Phase II Type 2 Diabetes clinical trial database was created and reviewed using standard CDM workflows including data validation, discrepancy identification, query management, and SQL-based data review.

## Study Summary

* Study Type: Phase II Clinical Trial
* Indication: Type 2 Diabetes
* Subjects: 50
* Sites: 5
* Visits: Baseline, Week 4, Week 8

## Clinical Domains Included

* DM (Demographics)
* MH (Medical History)
* CM (Concomitant Medications)
* AE (Adverse Events)
* LB (Laboratory Data)
* VS (Vital Signs)

## Project Activities

### Data Review

Reviewed clinical trial datasets to identify missing, inconsistent, duplicate, and out-of-range values.

### Validation Checks

Defined validation rules to detect:

* Missing mandatory fields
* Duplicate Subject IDs
* Future dates
* Invalid date sequences
* Negative values
* Out-of-range laboratory values
* Unrealistic vital sign measurements

### Query Management

Documented identified discrepancies and generated queries for issue tracking and resolution.

### SQL Analysis

Performed SQL-based data review to identify:

* Missing values
* Duplicate records
* Future dates
* Negative values
* Laboratory data discrepancies
* Vital sign abnormalities
